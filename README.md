## ¿Qué es Git?
Git es un sistema de control de versiones que permite gestionar los cambios en el código fuente de un proyecto. 
Con Git, los desarrolladores pueden trabajar en equipo, crear ramas para nuevas funcionalidades, y mantener un historial completo de los cambios realizados. 
Es una herramienta distribuida, lo que significa que cada colaborador tiene una copia completa del repositorio. 
Git facilita la colaboración, la revisión de código y la resolución de conflictos. 
Además, se integra con plataformas como GitHub para compartir proyectos en línea. 
Es ampliamente utilizado en la industria del software por su eficiencia y flexibilidad. 
Permite revertir errores, comparar versiones y mantener un flujo de trabajo ordenado. 
En resumen, Git es esencial para cualquier desarrollador moderno.

## 10 comandos de Git explicados
1. `git init` → Inicializa un repositorio Git.
2. `git add .` → Agrega todos los archivos al área de preparación.
3. `git commit -m "mensaje"` → Guarda los cambios con un mensaje.
4. `git status` → Muestra el estado actual del repositorio.
5. `git log` → Muestra el historial de commits.
6. `git branch` → Lista las ramas existentes.
7. `git checkout -b nombre` → Crea y cambia a una nueva rama.
8. `git merge rama` → Une una rama con otra.
9. `git push` → Sube los cambios al repositorio remoto.
10. `git pull` → Descarga los cambios del repositorio remoto.

## Flujo de trabajo recomendado
1. Crear una rama por funcionalidad.
2. Hacer commits frecuentes y descriptivos.
3. Subir los cambios a GitHub.
4. Crear Pull Requests para revisión.
5. Resolver conflictos si aparecen.
6. Hacer merge a la rama principal.
